    Class MoonTimes

    Observer-aware moon snapshot — exposes alt/az, rise/set, arc/track, peak, and topocentric angular diameter.

    The moon's nightly path varies meaningfully (declination drifts ~5°/day across the lunar month), so arc is the recommended way to render the moon's actual track over a night, rather than relying on a single peak point.

    Since

    0.2.0

    Example

    const moon = new MoonTimes({
      latitude: 40.7128,
      longitude: -74.0060,
      timezone: "America/New_York",
      time: new Date("2026-04-30T22:00:00Z"),
    });
    moon.position();         // { altitude, azimuth }
    moon.nextRise();         // { from, fromTz, ... }
    moon.arc({ samples: 60 });

    arc

    • arc(options?: { from?: Date; samples?: number; to?: Date }): IMoonArcSample[]

      Sample the moon's altitude/azimuth across an interval. Default window is nextRise() → following nextSet(); default sample count is 60. Each sample's date is monotonically increasing.

      For circumpolar moons (the moon never sets — rare but happens at extreme latitudes when the moon's declination matches), this returns an empty array; use dailyTrack instead.

      Parameters

      • Optionaloptions: { from?: Date; samples?: number; to?: Date }

        samples, from, to. Omitted fields default to the visible-arc window.

      Returns IMoonArcSample[]

      Array of { date, altitude, azimuth } samples.
